# Agent Operating Model

How multiple Claude Code / Codex agents collaborate on this repository without
conflict. The governing principle: **non-overlapping file ownership**. Two agents
must never edit the same workflow or generator at the same time.

## Roles and ownership (one owner per domain)

| Agent | Owns (allowed) | Must not touch |
|---|---|---|
| **Macro Intelligence** | `tools/build-economic-intelligence`, `forecast-consensus`, `reaction-intelligence`, `regime-intelligence`, `tools/providers/economic-calendar/*` | workflows, CSS, social, SEO |
| **Editorial** | article/brief/outlook generators, `editorial-quality`, `editorial-personas`, narrative scorers | workflows, providers, social ledgers |
| **Visual Intelligence** | SVG renderers, chart narratives, evidence-rail CSS within scope | generators' prose, workflows, providers |
| **Workflow** | `.github/workflows/*` only | tool/source logic, generators |
| **Validator** | `tools/check-*.js` | the code the checks validate (cannot weaken gates to pass) |
| **Distribution** | social previews, approval queue, posting ledger, distribution plan | publishing engine, generators |
| **UI** | public surface rendering (page generators, calendar JS) | providers, social, workflows |
| **SEO** | sitemaps, meta, canonical, indexing | generators, social, workflows |
| **Integration Governor** | merges, resolves conflicts, runs the full suite | (reviews everything, owns nothing exclusively) |
| **Safety** | reviews anti-fabrication, social, publishing, secrets | (review/veto only; does not implement features) |

The **Integration Governor** merges and validates. The **Safety Agent** reviews
anti-fabrication, social preview-only, Telegram gating, and secrets before any
push.

## When to use parallel agents

- Tasks touch **disjoint** directories (e.g. one agent on `tools/providers/`,
  another on `css/`). Confirm zero file overlap first.
- A phase splits cleanly into independent missions with separate validators.

## When NOT to use parallel agents

- Any chance two agents edit the same workflow, generator, or artifact.
- Sequential dependencies (B needs A's output) — run serially.
- Small phases — a single focused agent is cheaper and safer than coordination.

## Avoiding file conflicts

- Each mission declares its **allowed** and **forbidden** directories up front.
- Shared artifacts (`data/**`) are regenerated by their owning workflow, never
  hand-edited by multiple agents.
- The Integration Governor owns the final `git add` and resolves any overlap.

## Reducing token usage

- Audit narrowly: read only the files the mission will touch.
- Prefer one focused agent over many cold-start spawns; each spawn re-derives
  context. Do not spawn agents the task does not require.
- Reuse existing validators and patterns rather than re-deriving them.

## Splitting phases into focused missions

Decompose a large phase by **domain ownership** (above), give each mission a
single owner, an explicit file scope, and its own validator set, then let the
Integration Governor merge and run the full affected suite.
